# Ball Route — 中式八球白球走位分析工具 设计文档

## 概述

Ball Route 是一个面向进阶/竞技台球玩家的 Web 工具，核心功能是分析中式八球中白球的走位路线与杆法之间的映射关系。支持两种分析模式：**正向模拟**（选择杆法 → 预览轨迹）和**反向分析**（指定走位目标 → 推算杆法）。

**目标用户：** 有一定基础的台球爱好者和竞技玩家，已了解基本杆法概念，需要精确的可视化工具来验证走位判断、练习路线规划。

**技术方案：** 纯前端 Web 应用（TypeScript + HTML5 Canvas），自研物理引擎，零后端依赖。

## 核心功能

### 1. 正向模拟（杆法 → 轨迹）

用户设定球局（白球、目标球、其他障碍球的位置），选择击球方向、杆法和力度，系统实时计算并渲染白球的预测轨迹，包括：

- 白球击打目标球前的行进路线
- 碰撞后白球和目标球的分离路线
- 库边反弹路线（含侧旋修正）
- 白球最终停留位置

### 2. 反向分析（目标位置/路线 → 杆法）

用户指定白球击打目标球后的期望走位，系统反推可能的杆法组合：

- **点击模式：** 用户点击球台上白球期望停留的目标位置，系统计算达到该位置所需的杆法和力度组合
- **画线模式：** 用户在球台上画出白球期望的行进路径（包括库边反弹），系统拟合该路径并推算对应杆法

反向分析输出多个备选方案，按偏差排序，每个方案包含杆法类型、力度、预计偏差。

### 3. 球局管理

- **自由摆球：** 在球台上任意拖放球的位置
- **预设球局：** 内置常见走位练习场景（定点走位、翻袋走位等）
- **保存/加载：** 用户可保存自定义球局

## 支持的杆法

### 基本杆法
| 杆法 | 击球点 | 效果 |
|------|--------|------|
| 中杆 | 球心 | 白球撞击后滑行，最终自然滚动 |
| 高杆 | 球心上方 | 前旋（topspin），白球撞击后跟进 |
| 低杆 | 球心下方 | 回旋（backspin），白球撞击后缩回 |
| 左塞 | 球心左侧 | 左侧旋，影响库边反弹角度 |
| 右塞 | 球心右侧 | 右侧旋，影响库边反弹角度 |

### 组合杆法
高杆+左塞、高杆+右塞、低杆+左塞、低杆+右塞，以及各种中间位置的组合。

### 力度分级
轻、中、重三档基础分级，加上连续力度滑块（0%-100%），力度影响球速、旋转衰减距离和最终走位。

## 系统架构

```
用户操作 → InteractionManager → CueAnalyzer → PhysicsEngine → 计算结果 → TableRenderer → 画面更新
```

### 模块划分

| 模块 | 职责 | 关键接口 |
|------|------|----------|
| **PhysicsEngine** | 球体运动、旋转、摩擦、碰撞、库边反弹的数值模拟 | `simulate(state, params): SimulationResult` |
| **CueAnalyzer** | 正向模拟和反向推算的高层逻辑 | `forward(cueParams): Trajectory`、`reverse(target): CueOption[]` |
| **TableRenderer** | Canvas 2D 渲染球台、球、轨迹、辅助标记 | `render(state, trajectory)` |
| **InteractionManager** | 用户交互：拖放、画线、点击、面板控制 | 事件驱动，发射用户意图事件 |
| **ScenarioManager** | 预设球局存储与加载，自定义球局的 localStorage 持久化 | `load(id): TableState`、`save(state)` |

### 技术栈

- **语言：** TypeScript（严格模式）
- **构建：** Vite
- **渲染：** HTML5 Canvas 2D API
- **状态管理：** 简单的事件驱动架构，无框架依赖
- **持久化：** localStorage（球局存储）
- **未来演进：** 物理引擎核心可替换为 Rust → WebAssembly

## 物理引擎设计

### 运动模型

球体状态由以下参数描述：

- **位置** (x, y)：球心在球台坐标系中的位置
- **线速度** (vx, vy)：球心平移速度
- **角速度** (wx, wy, wz)：三轴旋转速度
  - wx：绕 x 轴旋转（前进/后退方向的滚转）
  - wy：绕 y 轴旋转（侧向滚转）
  - wz：绕 z 轴旋转（竖轴自转，即"塞"）

### 摩擦模型

球与台呢之间存在两种摩擦状态：

1. **滑动摩擦（sliding friction）：** 当球的线速度与角速度不匹配时（即接触点有相对滑动），滑动摩擦力改变线速度和角速度，逐渐使球过渡到纯滚动状态。这是杆法走位的核心机制。
2. **滚动摩擦（rolling friction）：** 纯滚动状态下，滚动摩擦使球逐渐减速直到停止。系数远小于滑动摩擦。

关键物理过程——**滑动→滚动转换**：不同杆法决定了初始角速度与线速度的比值，从而决定了滑动阶段的持续时间和方向，这直接影响白球在碰撞后的走位路径。

### 模拟参数

采用固定时间步长的数值积分（Euler 方法，MVP 阶段足够精确），时间步长 dt = 0.5ms（0.0005s）。每帧最多模拟 20 秒（40000 步），超时或所有球速度低于阈值（0.1mm/s）时终止模拟。

### 碰撞模型

**球-球碰撞：**
- 基于动量守恒和能量守恒（含恢复系数 e ≈ 0.95）
- 碰撞时旋转的传递：侧旋和前/后旋在碰撞时部分传递给目标球
- 薄厚比（cut angle）影响分离角

**球-库边碰撞：**
- 基础反射角 + 侧旋修正：带塞的球触库后反弹角度会偏离理想反射角
- 库边橡胶的恢复系数（≈ 0.8），导致触库后速度损失
- 库边接触时的摩擦导致旋转变化

**球-袋口：**
- 简化处理：球心进入袋口有效区域即判定落袋
- 角袋和中袋的有效区域尺寸不同（角袋略大于中袋）

### 杆法映射

击球参数用 `(offset_x, offset_y, force)` 描述：

- `offset_x`：击球点水平偏移（-1 到 1），负值为左塞，正值为右塞
- `offset_y`：击球点垂直偏移（-1 到 1），负值为低杆，正值为高杆
- `force`：击球力度（0 到 1）

击球点偏移量映射为初始角速度：
- 高/低偏移 → wx（前旋/回旋）
- 左/右偏移 → wz（侧旋）
- 偏移量受安全区域限制（半径约 0.7R，避免滑杆 miscue）

### 中式八球物理参数

| 参数 | 值 |
|------|-----|
| 球台内径 | 2540mm × 1270mm |
| 球直径 | 57.15mm |
| 球质量 | 170g |
| 滑动摩擦系数 | 0.2 |
| 滚动摩擦系数 | 0.01 |
| 球-球恢复系数 | 0.95 |
| 库边恢复系数 | 0.8 |
| 角袋口宽度 | ~84mm（约 1.47 倍球径） |
| 中袋口宽度 | ~76mm（约 1.33 倍球径） |

### 反向推算算法

反向分析（给定目标位置，推算杆法）采用优化搜索策略：

1. 根据白球、目标球位置和击球方向，确定碰撞后的基本分离角
2. 在杆法参数空间 `(offset_x, offset_y, force)` 中进行网格搜索
3. 对每组参数运行正向模拟，计算白球终点与目标位置的偏差
4. 对偏差最小的候选方案做局部优化（梯度下降或 Nelder-Mead）
5. 返回 Top-N 个偏差最小的方案

画线模式下，额外约束轨迹路径的形状匹配（不仅终点，还要路径经过指定库边）。

## 界面设计

### 布局方案

采用**横屏全屏沉浸式**布局，适配手机横屏和小程序嵌入：

- **球台全屏展示：** 球台占据整个画面，最大化可视面积
- **底部浮层工具栏：** 半透明悬浮，包含迷你击球点选择器、力度滑条、功能按钮
- **右侧滑出结果面板：** 分析结果从右侧滑入（占 35% 宽度），球台自动左移，毛玻璃背景
- **TV 直播风格：** 四角暗角（vignette）、左上角 Logo 水印、底部信息条叠加、中央顶灯光晕

### 球台渲染细节

- **木框：** 多层木纹纹理 + 金色镶嵌装饰线
- **台呢：** 细微布纹纹理 + 暖色灯光漫反射
- **袋口：** 角袋分层结构（外圈袋口 + 内圈深洞 + 库边弧形收口），中袋半椭圆形，均有深度渐变
- **球体：** 3D 光泽渐变 + 高光亮斑 + 台面投影，花球带条纹渲染
- **钻石标记：** 菱形金属渐变镶嵌物

### 交互方式

| 操作 | 触发 | 效果 |
|------|------|------|
| 拖放球 | 长按并拖动 | 移动球到任意位置 |
| 设定击球方向 | 从白球向外拖动 | 显示瞄准线 |
| 选择击球点 | 点击/拖动击球点选择器 | 设定杆法 |
| 调节力度 | 拖动力度滑条 | 设定击球力度 |

### 响应式适配

- **手机横屏（主要）：** 全屏沉浸式，底部浮层 + 右侧滑出面板
- **手机竖屏：** 球台在上，底部 Tab 切换杆法/结果/球局面板
- **桌面浏览器：** 球台居中，可选三栏布局
- **小程序嵌入：** 与手机横屏布局一致，通过 `<web-view>` 加载

## 数据模型

```typescript
interface TableConfig {
  width: number;         // 球台内径宽度 (mm)，默认 2540
  height: number;        // 球台内径高度 (mm)，默认 1270
  ballDiameter: number;  // 球直径 (mm)，默认 57.15
  pocketRadius: {        // 袋口半径 (mm)
    corner: number;      // 角袋，默认 42
    side: number;        // 中袋，默认 38
  };
}

interface TableState {
  balls: Ball[];
  tableConfig: TableConfig;
}

interface Ball {
  id: number;
  x: number;
  y: number;
  type: 'cue' | 'solid' | 'stripe' | 'eight';
  number: number;
  pocketed: boolean;
}

interface CueParams {
  aimAngle: number;      // 瞄准角度（弧度）
  offsetX: number;       // 击球点水平偏移 (-1 ~ 1)
  offsetY: number;       // 击球点垂直偏移 (-1 ~ 1)
  force: number;         // 力度 (0 ~ 1)
}

interface SimulationResult {
  trajectory: TrajectoryPoint[];
  finalPosition: { x: number; y: number };
  events: CollisionEvent[];
}

interface TrajectoryPoint {
  x: number;
  y: number;
  t: number;             // 时间戳
  vx: number;
  vy: number;
  spin: { wx: number; wy: number; wz: number };
}

interface CollisionEvent {
  type: 'ball-ball' | 'ball-cushion' | 'ball-pocket';
  t: number;             // 发生时间
  ballId: number;        // 主体球 ID
  targetId?: number;     // 碰撞目标球 ID（ball-ball 时）
  position: { x: number; y: number };
}

interface CueOption {
  params: CueParams;
  trajectory: TrajectoryPoint[];
  deviation: number;     // 与目标位置的偏差 (mm)
  label: string;         // 杆法描述（如"高杆+右塞"）
}
```

## 预设球局

MVP 阶段内置以下练习场景：

1. **直线跟进/缩回：** 白球和目标球在一条直线上，练习高杆跟进和低杆缩回
2. **定点走位 - 近距离：** 短距离击球后白球走位到指定区域
3. **定点走位 - 远距离：** 长台击球后的走位控制
4. **一库走位：** 白球碰撞后需要打一个库边到达目标区域
5. **两库走位：** 白球碰撞后需要打两个库边
6. **塞球走位：** 练习侧旋对库边反弹角的影响

## 错误处理

- **滑杆检测：** 击球点偏移超出安全区域时，提示用户"该击球点可能导致滑杆"
- **无解提示：** 反向分析时，若没有杆法组合能使白球到达目标位置（如被障碍球阻挡），给出明确提示
- **物理异常：** 模拟过程中球速异常（数值发散）时自动终止并提示

## 演进路线

1. **MVP（当前）：** 纯 TypeScript 物理引擎，核心正向/反向分析，横屏 UI
2. **中期优化：** Web Workers 分离物理计算线程，动画轨迹回放，更多预设球局
3. **长期目标：** Rust → WASM 物理引擎，WebGPU 渲染，多台球类型支持（九球、斯诺克）
